# リモデル業務分析

> 顧客による解説動画トランスクリプト（2026-03-13）をもとに整理

---

## 1. 現行業務の把握（As-Is）

### 全体タイムライン（病院建設プロジェクト: 3〜5年）

```
調査構想 → 基本計画 → 基本設計（1年） → 実設計 → 工事業者選定（半年） → 工事（2年） → 竣工・開院
```

基本設計終了時に提示したもの
元々出したものに対して何がどう変わって金額が変わったか提示は必要。
元々出したものはExcel出力でアーカイブする運用

### 業務フロー

#### Phase 1: 現有品調査
- 既存病院の図面をもとに部署マスター・部屋マスターを作成
- 現地に**紙のリスト**を持参し、機器にラベル貼付・写真撮影
- 帰社後にデータを**事後入力**（品目名・数量・ラベル番号）
- この段階では大分類・中分類は未入力

#### Phase 2: ヒアリング（部署ごと・約1時間）
- 持参物: 写真集、新病院図面、現有品リスト
- **事前準備**: Excelリストを機器種別順に並べ替え、ヒアリング用カラムだけ表示
- **事前配布**: 先生方に紙を渡し、購入区分（新規/更新/移設等）を事前記入してもらう場合あり
- ヒアリング中の操作:
  - **新規**: Excelで行挿入→品目名・部屋名を手書きまたは入力
  - **更新**: 対象行をコピー→元を「廃棄予定」、新行を「更新」に変更（台帳情報の不要列を削除する手作業が発生）
  - **増設**: 必要数だけ行を追加
  - **移設**: 新病院の部屋名を記入
- **課題**: 行の挿入・コピー・削除で台帳紐付け列まで複製される、並び替えが手間、手書きメモの事後入力

#### Phase 3: 設備リスト作成（建築向け）
- ヒアリング後、設備諸条件リスト（電気容量・給排水・特殊設備等）を作成
- 品目ごとにカタログから電気容量等を**一件一件手作業で調査・入力**（数千行）
- レイアウト図面と紐付けて設計事務所へ提出

#### Phase 4: 見積もり依頼
- 品目を取扱メーカー単位でグルーピング（例: 生理検査機器一式）
- グループごとにリストを作成→メーカーにメール送付
- 検査部の実験台のように「一式」で図面付き依頼もある

#### Phase 5: 見積もり登録
- メーカーから届いた見積もりの金額をExcelリストに手入力
- 一式金額の場合は「上記に含む」等で記載（個別単価不明が多い）
- 親子関係の整理（親機器・子付属品・孫消耗品）は人手で判断
- 見積もりに含まれる消耗品・保守料を**カテゴリ分け**して按分対象から除外

#### Phase 6: 予算書・比較資料作成
- 部署ごとのサマリーをA3 1〜2枚に整理
- 複数メーカー比較資料の作成
- 減額案作成・優先順位付け
- すべてExcel上で手作業

#### Phase 7: 発注
- 発注グループを作成（見積依頼グループを統合する場合あり）
- 支払条件: 締め日/払い日、銀行振込/クレジット/現金、リースの場合は開始・終了日
- 院内決裁番号の管理
- 発注書の出力

#### Phase 8: 納品・検修
- 納品日登録（リモデルは数百〜数千件→一括登録が必要）
- QRラベル印刷（親+子の枚数）、検修書類準備
- **現地検修**: ラベル貼付→QR読取→シリアル番号入力→写真撮影→部屋名確認
- 現行は紙ベース、2000〜3000台規模では手書き検修は非現実的

#### Phase 9: 資産登録
- 会計区分の確認（有形資産/消耗品/保守契約）
- 固定資産番号は会計システムから取得→本システムに最終登録
- 厚労省の病院会計準則に基づくカテゴリ（医療機械/医療用具等、金額による区分変動あり）

## 2. ToBe業務フロー

```
┌─────────────────────────────────────────────────────────┐
│ 準備                                                       │
│  ① 施設マスター登録（施設名・病床数）                           │
│  ② 個別部署マスタ登録（共通マスタと紐付け）
├─────────────────────────────────────────────────────────┤
│ 現有品調査                                                   │
│  ③ モバイルで調査（QRラベル貼付→スキャン→写真撮影→品目登録）   │
│  ④ 原本の複製である編集リスト作成                                           │
├─────────────────────────────────────────────────────────┤
│ ヒアリング                                                   │
│  ⑤ ヒアリング用カラムプリセット適用                            │
│  ⑥ 部署・機器種別でソート表示                            │
│  ⑦ モーダルで新規品目を複数一括登録                            │
│  ⑧ 「更新」操作→自動で旧レコード=廃棄予定＋新レコード生成       │
│  ⑨ 新病院部屋名・区分（新規/更新/移設/増設）・コメント入力       │
│  ⑩ 事前に先生方へ閲覧権限付与→購入区分を事前入力              │
├─────────────────────────────────────────────────────────┤
│ 設備条件整理                                                 │
│  ⑪ データリンク: 資産マスターIDから設備条件を一括自動入力        │
│  ⑫ 設備諸条件リスト出力（レイアウト用）                        │
├─────────────────────────────────────────────────────────┤
│ 見積もり                                                     │
│  ⑬ 見積依頼グループ作成（最小単位でグルーピング）               │
│  ⑭ 複数依頼先（ベンダー）の登録                               │
│  ⑮ 見積依頼書出力・送付                                       │
│  ⑯ 見積登録: OCR/手入力→AI親子判定→カテゴリ区分→按分         │
│  ⑰ レコード置換（見積データで編集リストを一括更新、コメント保持） │
├─────────────────────────────────────────────────────────┤
│ 予算・比較                                                   │
│  ⑱ Excel出力: 複数ベンダー比較資料、部署別サマリー             │
│  ⑲ 減額案・優先順位付け（Excel上で対応）                       │
├─────────────────────────────────────────────────────────┤
│ 発注                                                         │
│  ⑳ 発注グループ作成（見積依頼グループを統合可）                 │
│  ㉑ 発注条件登録（支払条件・リース・院内決裁番号）               │
│  ㉒ 発注書プレビュー→確認→登録→出力/メール送付                │
├─────────────────────────────────────────────────────────┤
│ 納品・検修                                                   │
│  ㉓ 納品日一括登録                                             │
│  ㉔ QRラベル・検修書出力                                       │
│  ㉕ モバイル検修（QRスキャン→シリアル入力→写真撮影→部屋確認）  │
├─────────────────────────────────────────────────────────┤
│ 資産登録                                                     │
│  ㉖ 会計区分確認（勘定科目）                                   │
│  ㉗ 固定資産番号登録（会計システム連携）                         │
│  ㉘ 台帳登録完了                                               │
└─────────────────────────────────────────────────────────┘
```

**通常購入との棲み分け**:
- リモデル用「編集リスト」は通常購入リストとは**別管理**
- タスク管理でリモデル/通常購入は区分されているため、発注以降は共通フロー可
- 通常購入で年度内に購入・廃棄された分は、年度末にリモデル編集リストへ手動反映（リアルタイム連携は不要）

## 3. 本システムに取り込むべき機能

### A. リモデル編集リスト（中核機能）

| # | 機能 | 詳細 |
|---|------|------|
| A1 | リモデル専用編集リスト | 通常購入リストとは独立。プロジェクト期間（3〜5年）を通じて使用 |
| A2 | カラムプリセット（ブックマーク） | 「ヒアリング用」「設備条件用」等、用途別に表示カラムを切替 |
| A3 | 柔軟なソート・フィルター | 部署別、部屋別、機器種別、購入区分別 |
| A4 | 購入区分管理 | 新規/更新/移設/増設/廃棄予定 の5区分 |
| A5 | 執行希望年度 | 「2026年度更新」等、年度管理カラム |

### B. ヒアリング支援

| # | 機能 | 詳細 |
|---|------|------|
| B1 | 新規品目モーダル（複数一括登録） | モーダルから品目を複数選択して一括登録。品目名だけでもOK（メーカー・型番は後入力可） |
| B2 | 更新操作の自動化 | 「更新」選択→旧レコードを「廃棄予定」に自動変更＋新レコード自動生成（品目コピー、台帳情報は除外） |
| B3 | 増設操作 | 必要数のレコードを自動追加 |
| B4 | 更新元紐付け | 新レコードに「現有品調査番号○○の更新」が分かる参照リンク |
| B5 | 先生向け事前入力機能 | 部署限定の閲覧・編集権限を付与し、購入区分・部屋名を事前入力してもらう |
| B6 | リモデル用簡略遷移 | 優先順位入力など不要な項目はスキップ。確認画面遷移を省略 |

### C. 設備条件管理

| # | 機能 | 詳細 |
|---|------|------|
| C1 | 資産マスターに設備条件格納 | 電気容量、給排水、特殊設備、レイアウト表示要否、寸法等を品目/型番レベルで管理 |
| C2 | データリンク一括適用 | 選択レコードに対し、資産マスターIDから設備条件カラムを一括自動入力 |
| C3 | 設備諸条件リスト出力 | 建築向けリスト（図面番号紐付き）のExcel出力 |
| C4 | マスター粒度の3段階管理 | 品目レベル / メーカーレベル / 型番レベル で設備条件の精度が異なる |

### D. 見積もり関連

| # | 機能 | 詳細 |
|---|------|------|
| D1 | 見積依頼グループ（最小単位） | 品目種別・メーカー単位でグルーピング。何度でも作り直し可 |
| D2 | 複数依頼先の登録 | 1グループに対し複数ベンダーへ見積依頼 |
| D3 | AI親子判定 | 見積明細の親/子/孫/その他を自動分類（SHIP確認フラグ付き） |
| D4 | カテゴリー区分 | 医療機械/消耗品/保守料/設置費/工事費等。按分対象の除外判定に使用 |
| D5 | 親子紐付け | 親-子はマスト、親-孫は任意。紐付けUI必要 |
| D6 | 価格按分 | 親+子セットでの按分ルール（定価不明の場合の対応含む） |
| D7 | レコード一括置換 | 見積依頼グループ単位で、編集リストのレコードを見積データで上書き。ただしコメント・要望は保持 |
| D8 | 部屋名追加入力 | レコード置換後、増えたレコードに部屋名を追加設定 |

### E. 発注関連

| # | 機能 | 詳細 |
|---|------|------|
| E1 | 発注グループ作成 | 見積依頼グループを統合して発注グループ化 |
| E2 | 発注条件 | 締め日/払い日、支払方法（振込/クレジット/現金）、○日サイト、コメント欄 |
| E3 | リース対応 | 開始日・終了日 |
| E4 | 院内決裁番号 | 発注登録前に入力可能な欄 |
| E5 | 発注書プレビュー確認 | 登録前にプレビュー表示→修正可→確定→出力/メール |
| E6 | レンタル区分は不要 | 発注画面のレンタルは契約管理側の機能のため除外 |
| E7 | 手形廃止対応 | 手形を選択肢から削除（2027年廃止予定） |

### F. 納品・検修

| # | 機能 | 詳細 |
|---|------|------|
| F1 | 納品日一括登録 | まとめて同じ日付を入力→個別に修正可 |
| F2 | QRラベル印刷 | 親+子の数量分のラベル出力 |
| F3 | 検修書出力 | 部屋名まで入った状態で出力 |
| F4 | モバイル検修 | タスク管理から検修対象を選択→モバイルで実施。QRスキャン→シリアル入力→写真撮影→部屋名修正 |
| F5 | 検修対象絞込み | 発注済み＋検修可能なもののみモバイル画面に表示 |

### G. 資産登録

| # | 機能 | 詳細 |
|---|------|------|
| G1 | 会計区分表示 | 勘定科目（有形資産/消耗品/保守契約）をデフォルト設定済みで表示。経理が最終確認・修正 |
| G2 | 固定資産番号登録 | 会計システムから発番された番号を最終入力して台帳登録完了 |
| G3 | 金額による区分自動判定 | 固定資産計上基準額を下回る場合の自動区分変更（医療用具→消耗品等） |

### H. データ出力（第2フェーズ）

| # | 機能 | 詳細 |
|---|------|------|
| H1 | 編集リストExcel出力 | 任意カラム指定でのExcel出力 |
| H2 | 部署別サマリー出力 | A3 1〜2枚のサマリー形式 |
| H3 | 複数ベンダー比較出力 | 同一品目の複数見積を横並び比較 |
| H4 | 減額案作成支援 | 優先順位・金額でのフィルタリング→出力 |
| H5 | 見積データベースExcel出力 | 全見積データ（親子関係・マスター紐付き済み）の出力 |
