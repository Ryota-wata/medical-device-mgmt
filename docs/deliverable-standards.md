# 成果物作成基準（システム開発①）

検収基準書（`docs/検収基準書.xlsx`）に基づく、各納品物の作成基準。
成果物を生成・更新する際は、該当する納品物の基準を**すべて満たす**こと。

---

## 1. 画面設計書

- **形式**: Excel（1ファイル、画面ごとにシート）
- **対象**: 全63画面

### 必須構成（各画面シート）
| 構成要素 | 必須内容 |
|---|---|
| 画面レイアウト図 | スクリーンショット + No.マーカー |
| 項目定義表 | 項目名・データ型・桁数・必須/任意 |
| 操作仕様 | 各操作のトリガー・動作・遷移先 |
| バリデーション | 入力項目ごとのルール・エラーメッセージ |
| 権限定義 | ロール別の表示/非表示・操作可否 |

### 品質基準
- 画面間で同一データの項目名・データ型が統一されていること
- 設計書の記載と実装画面が一致すること
- 目次シートに全63画面が列挙されていること

---

## 2. 画面遷移設計書

- **形式**: Excel
- **対象**: 全63画面間の遷移

### 必須構成
| 構成要素 | 必須内容 |
|---|---|
| 遷移定義 | 遷移元・遷移先・トリガー（ボタン名等）・遷移条件 |
| ロール別アクセス | 6ロールそれぞれのアクセス可否 |
| 遷移図 | 全画面の遷移関係を図示 |

### 品質基準
- 全63画面が遷移図に含まれ、孤立した画面がないこと
- ログイン画面から全画面に到達可能なパスが存在すること
- 実装の画面遷移と設計書が一致すること

---

## 3. DB設計書

- **形式**: Word + PlantUML/SVG
- **対象**: 94テーブル + ビュー

### 必須構成
| 構成要素 | 必須内容 |
|---|---|
| ER図 | PlantUML/SVG形式、全テーブルのリレーション |
| テーブル定義 | カラム名・データ型・NOT NULL等の制約 |
| 外部キー定義 | テーブル間のリレーション |
| インデックス定義 | 各テーブルのインデックス |

### 品質基準
- 全94テーブル+ビューの定義が存在すること
- 外部キーとER図のリレーション線が一致すること
- 画面設計書の項目がDBカラムにマッピング可能であること
- 主要テーブルが第3正規形を満たすこと

---

## 4. API設計書

- **形式**: Word
- **対象**: 全API機能（不要4件除く）

### 必須構成（各API）
| 構成要素 | 必須内容 |
|---|---|
| エンドポイント | URL・HTTPメソッド（GET/POST/PUT/DELETE） |
| リクエスト定義 | パラメータ名・型・必須/任意 |
| レスポンス定義 | ボディ項目名・型 |
| 認証・認可 | 認証方式・必要ロール |
| エラーコード | HTTPステータス・エラーコード・メッセージ |

### 品質基準
- 参照するテーブル・カラムがDB設計書に存在すること
- 画面設計書の主要操作（CRUD）に対応するAPIが網羅されていること
- 全API一覧と各API設計書が対応していること

---

## 5. ソフトウェア一式

- **形式**: ZIP
- **対象**: システム開発①スコープ（全63画面）

### 必須構成
| 構成要素 | 必須内容 |
|---|---|
| ZIPファイル | プロジェクトのディレクトリ構成・ソースコード一式 |
| README | 環境構築手順・起動手順 |
| 全63画面 | 表示・基本操作が可能 |
| 認証 | ログイン機能 |
| 権限制御 | 6ロール別の表示/非表示・操作可否 |
| レスポンシブ | PC（1920x1080）・タブレット（1024x768）・スマホ（375x812）対応 |

### 品質基準
- `npm run build` がエラーなく完了すること
- TypeScript型エラーが0件であること
- Chrome・Edge・Safariの最新版で表示崩れがないこと
- ログイン〜各機能の基本操作フローが動作すること

---

## 6. テスト仕様書及びエビデンス

- **形式**: Excel
- **対象**: 全63画面 + 主要業務フロー

### 必須構成
| 構成要素 | 必須内容 |
|---|---|
| テスト項目 | テスト項目・操作手順・期待結果 |
| 実施結果 | 実施日・実施者・結果（合格/不合格）・エビデンス |
| 画面表示テスト | 全63画面の表示テスト |
| 結合テスト | 主要業務フロー（リモデル申請・現有品調査等） |
| 権限テスト | 6ロール別アクセス制御 |
| 異常系テスト | バリデーションエラー・権限エラー等 |

### 品質基準
- 全テスト項目が実施済み（実施率100%）であること
- 不合格項目がある場合は対応状況（修正済み/対応予定等）が記録されていること

---

## 7. 運用ドキュメント

- **形式**: Word/PDF

### 必須ドキュメント
| ドキュメント | 必須内容 |
|---|---|
| 障害時復旧手順書 | 想定障害パターン（サーバー停止・DB障害等）と具体的な復旧手順 |
| VPNClient導入手順書 | インストール〜設定〜接続確認までの手順 |

### 品質基準
- スクリーンショットまたは説明図が含まれていること
- 手順書に従って操作した場合に目的（復旧/VPN接続）が達成可能であること

---

## 成果物間の整合性ルール

成果物は単独ではなく相互に参照関係を持つ。以下の整合性を常に維持すること。

| 参照元 | 参照先 | 整合ポイント |
|---|---|---|
| 画面設計書 | DB設計書 | 画面項目 → DBカラムのマッピング |
| 画面設計書 | API設計書 | 画面操作 → API呼び出しの対応 |
| 画面設計書 | 画面遷移設計書 | 画面一覧・遷移先の一致 |
| API設計書 | DB設計書 | 参照テーブル・カラムの存在 |
| テスト仕様書 | 画面設計書 | テスト対象画面の網羅（63画面） |
| テスト仕様書 | ソフトウェア | テスト結果と実装の一致 |
| ソフトウェア | 画面設計書 | 実装と設計の一致 |
