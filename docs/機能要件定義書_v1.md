## 1. システム概要

### 1.1 システム名称
医療機器管理システム

### 1.2 システム目的
医療機関における医療機器の資産管理、点検管理、貸出管理、修理管理を一元化し、業務効率化とトレーサビリティを実現する。

### 1.3 対象ユーザー（想定）
- ME技師（臨床工学技士）
- 医療機器管理責任者
- 病棟看護師
- 施設管理者
- システム管理者

## 4. データモデル概要

### 4.1 主要エンティティ

| エンティティ | 説明 | 主要属性 |
|-------------|------|----------|
| Asset | 資産 | QRコード、施設、棟、階、部門、部署、カテゴリ、大分類、中分類、品目、メーカー、型式、数量、寸法 |
| InspectionTask | 点検タスク | 資産ID、点検種別、メニューID、次回点検日、ステータス |
| InspectionMenu | 点検メニュー | 名称、対象分類、点検種別、周期、点検項目 |
| LendingDevice | 貸出機器 | QRラベル、品目、ステータス、貸出日、返却予定日 |
| User | ユーザー | ID、名前、権限 |
| Facility | 施設 | 施設コード、施設名、棟、階、部門、部署 |

### 4.2 マスタデータ

| マスタ | 説明 |
|--------|------|
| 施設マスタ | 病院・施設の階層構造 |
| 資産分類マスタ | カテゴリ→大分類→中分類→品目の階層 |
| 部署マスタ | 部門・部署情報 |
| ユーザーマスタ | ユーザー・権限情報 |

## 5. 画面遷移概要

### 5.1 主要フロー

```
ログイン → メイン画面
              ├→ 資産管理 → 検索結果 → 詳細 → 編集
              ├→ 現地調査 → 場所選択 → 調査入力 → 履歴
              ├→ QR管理 → QR発行 → 印刷
              ├→ 見積・発注 → 見積一覧 → OCR確認 → マッチング → 登録
              ├→ 点検管理 → タスク一覧 → 点検実施
              ├→ 貸出管理 → 機器一覧 → 貸出/返却
              └→ マスタ管理
```
