# taniguchi 配下の管理方針

この配下には、谷口さん管理の正本・補助資料・API設計書生成基盤をまとめる。

## 構成
- `機能要件.md`
  - 画面要件と DB 設計の正本
- `db/`
  - `db-schema.puml` と `db-schema.svg` を置く DB 設計の正本フォルダ
- `api/`
  - API 設計書一式
- `docs/`
  - 差分案や設計メモ
- `scripts/`
  - API 設計書生成スクリプトと spec
- `work/`
  - PoC や一時検証物の退避先

## 運用
- 正本として扱う文書は、この配下のものを参照する。
- API 設計書の下書き生成は `scripts/` の Word 生成基盤を利用する。
- `api/参考_作業用` は安全な下書き出力先とし、正本更新は明示指示時のみ行う。
