# スクリーンショットフィードバック対応スキル

顧客が画面モックをレビューし、スクリーンショットにコメントや注釈を付けて送ってきた場合に、そのフィードバックを解析して既存モックを改修するスキルです。

## スクリーンショット格納場所

```
/Users/watanaberyouta/Desktop/FBスクリーンショット/
```

## 対応フロー

### Step 1: スクリーンショットの確認

FBスクリーンショットフォルダ内の画像を確認する。

```bash
ls -la "/Users/watanaberyouta/Desktop/FBスクリーンショット/"
```

### Step 2: 画像の読み込み

フォルダ内のすべての画像をReadツールで読み込む。

```
Read: /Users/watanaberyouta/Desktop/FBスクリーンショット/xxx.png
```

### Step 3: 画面要素の解析

スクリーンショットから以下を読み取る：

| 解析項目 | 内容 |
|---------|------|
| **ヘッダー/ボディ識別** | スクショ内のヘッダー部分とボディ部分を区別 |
| 画面レイアウト | カラム構成、配置、余白 |
| UI要素 | ボタン、テーブル、フォーム、タブ等 |
| 色・スタイル | 背景色、ボーダー、フォントサイズ |
| テキスト | ラベル、説明文、注釈 |
| コメント・注釈 | 赤枠、矢印、吹き出し等の指摘 |

### Step 4: 修正対象画面の確認

**重要**: 解析結果をユーザーに報告し、修正対象画面を確認する。

```
## スクリーンショット解析結果

### ヘッダー/ボディ識別
- ヘッダー部分: ...（修正要素の有無）
- ボディ部分: ...

### 画面構成
- ...

### 主なUI要素
- ...

### 修正が必要と思われるファイル
- ヘッダー: `components/layouts/Header.tsx`（該当する場合）
- ボディ: `app/xxx/page.tsx`

この内容で修正してよろしいですか？
```

### Step 5: UI/UXの検討

ユーザーの承認後、フィードバックを踏まえて実装方針を検討：

- **基本はフィードバックに忠実に実装**
- 技術的制約がある場合は代替案を提示
- 独自判断でUIを変更しない

### Step 6: 改修実施

1. 対象ファイルを読み込む
2. フィードバックに沿って修正
3. ビルドエラーがないか確認

### Step 7: 完了報告

改修内容をサマリーで報告。

## 使用例

### 基本的な使い方

```
FBスクリーンショットフォルダを見てフィードバック対応してください。
```

### 特定のスクリーンショットのみ

```
FBスクリーンショットフォルダの○○.pngを見て対応してください。
```

## 注意事項

1. **修正前の確認（省略可）**
   - ユーザーから「確認不要」「全て許可」等の指示がある場合は確認をスキップ
   - 指示がない場合のみ、修正対象画面をユーザーに確認してから着手

2. **フィードバック優先**
   - 顧客のフィードバックに忠実に実装
   - 独自にUI/UXを変更しない

3. **確認が必要なケース**
   - フィードバックが曖昧な場合
   - 技術的に実現困難な場合
   - 複数の解釈が可能な場合

4. **Gitプッシュ禁止**
   - ユーザーから許可が出るまでgit pushしないこと
   - ローカルでの確認が完了し、許可を得てからプッシュする

5. **吹き出し・説明文の扱い**
   - スクショ内の吹き出し・説明文は「実装対象ではなく解析用」
   - 吹き出しから機能を抽出して実装する：
     - プルダウンの選択肢
     - 画面遷移先
     - ボタンのアクション
     - ステータスの種類
   - 説明文をそのまま画面に表示しない

6. **ヘッダーとボディの識別**
   - スクショにはヘッダーが含まれている場合がある
   - スクショ内のヘッダー部分とボディ部分を識別すること
   - **ヘッダーに修正要素・コメントがある場合**：
     - 既存の`Header`コンポーネント（`components/layouts/Header.tsx`）を改修
     - スクショのヘッダー内容を既存Headerに反映させる
   - **ボディ部分の修正**：
     - 各画面の`page.tsx`を改修
   - 二重ヘッダーにならないよう注意（スクショのヘッダーをボディに実装しない）
