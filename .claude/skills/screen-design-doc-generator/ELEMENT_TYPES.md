# 要素種別定義

画面設計書で使用する要素種別の定義一覧です。

## 採番プレフィックス

| プレフィックス | カテゴリ | 説明 |
|---------------|----------|------|
| H | ヘッダー | 画面ヘッダー内の要素 |
| N | ナビゲーション | タブ、メニュー、パンくず等 |
| F | フィルター/検索 | 絞り込み、検索関連 |
| T | テーブル | テーブル、一覧表示 |
| B | ボタン | ボタン、アクション |
| L | ラベル/テキスト | 静的テキスト、見出し |
| I | 入力 | フォーム入力要素 |
| M | モーダル | モーダル、ダイアログ |
| C | カード | カード型コンポーネント |
| O | その他 | 上記に該当しないもの |

## データ元の分類

| 分類 | 説明 | 例 |
|------|------|-----|
| 固定値 | ハードコードされた値 | ラベル文字列、選択肢 |
| ユーザー入力 | ユーザーが入力する値 | テキストボックス |
| 画面内ステート | useState等の内部状態 | 選択状態、開閉状態 |
| API | APIから取得するデータ | API: GET /api/users |
| Store | グローバルステート | Zustand, Redux等 |
| Props | 親から渡される値 | 親コンポーネント |
| URL | URLパラメータ | クエリ文字列、パス |
| LocalStorage | ローカルストレージ | 保存された設定 |

## 命名規則

### 要素名の付け方

1. **ラベルがある場合**: ラベルテキストをそのまま使用
   - 例: `検索` → 「検索ボタン」

2. **プレースホルダーがある場合**: プレースホルダーを参考に命名
   - 例: `placeholder="キーワードを入力"` → 「キーワード入力欄」

3. **aria-label がある場合**: aria-labelを使用
   - 例: `aria-label="閉じる"` → 「閉じるボタン」

4. **上記がない場合**: 機能や位置から命名
   - 例: テーブル1行目の編集ボタン → 「行編集ボタン」
