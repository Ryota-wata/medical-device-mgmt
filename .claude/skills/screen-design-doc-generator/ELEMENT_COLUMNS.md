# 要素一覧カラム定義ルール

画面設計書の要素一覧を作成する際の12カラム判定ルール。

## カラム構成（12列）

| No | 要素名 | 要素種別 | データ型 | 桁数 | 必須 | 操作仕様 | 初期値 | バリデーション | エラーメッセージ | 権限 | 備考 |
|----|--------|----------|----------|------|------|----------|--------|----------------|------------------|------|------|

## データ型の判定ルール

ソースコード（page.tsx等）の型定義・input属性から判定する。

| データ型 | 判定基準 | TSX属性の例 |
|----------|----------|-------------|
| `string` | 通常のテキスト入力 | `type="text"`, `<textarea>` |
| `number` | 数値入力 | `type="number"`, `inputMode="numeric"` |
| `date` | 日付入力 | `type="date"`, DatePickerコンポーネント |
| `boolean` | ON/OFF切り替え | `type="checkbox"`, Switch, Toggle |
| `email` | メールアドレス入力 | `type="email"` |
| `password` | パスワード入力 | `type="password"` |
| `enum` | 選択肢（プルダウン/ラジオ） | `<select>`, SearchableSelect, RadioGroup |
| `file` | ファイルアップロード | `type="file"`, DropZone |
| `tel` | 電話番号 | `type="tel"` |
| `url` | URL | `type="url"` |
| `--` | データを持たない要素（ラベル、ボタン等） | 静的表示要素 |

## 桁数の判定ルール

### input要素から判定
- `maxLength="N"` → `最大N文字`
- `max={N}` → `最大N`（数値）
- `min={N} max={M}` → `N〜M`
- `pattern="..."` → パターンから推定

### 型から推定（maxLength指定なし）
| データ型 | デフォルト桁数 |
|----------|---------------|
| email | `最大256文字` |
| password | `最大128文字` |
| tel | `最大20文字` |
| date | `YYYY-MM-DD` |
| boolean | `--` |
| enum | `--` |
| file | `--` |
| url | `最大2048文字` |

### ビジネスロジックから推定
| フィールド用途 | 桁数 |
|----------------|------|
| 施設名 | `最大100文字` |
| 部署名 | `最大50文字` |
| 人名 | `最大50文字` |
| 資産管理番号 | `最大20文字` |
| 金額 | `整数12桁` |
| 数量 | `整数6桁` |
| 備考・コメント | `最大500文字` |

## 操作仕様の記載ルール

旧「機能説明」カラムを拡充。トリガー・動作・遷移先を含む。

### 記載フォーマット
```
{トリガー}→{動作/遷移先}
```

### 例
| 要素種別 | 操作仕様の例 |
|----------|-------------|
| ボタン | `クリック→ログイン処理を実行し、メニュー画面へ遷移` |
| リンク | `クリック→パスワード再設定画面へ遷移` |
| プルダウン | `選択変更→一覧テーブルをフィルタリング` |
| タブ | `クリック→対応するタブパネルを表示` |
| テキスト入力 | `入力→リアルタイムバリデーション実行` |
| テーブル行 | `行クリック→詳細画面へ遷移` |
| チェックボックス | `チェック切替→選択行をバッチ処理対象に追加` |
| ラベル | `（表示のみ）{表示内容の説明}` |
| 画像 | `（表示のみ）{画像の説明}` |

## エラーメッセージの判定ルール

ソースコード内のバリデーションロジックから抽出する。

### 探索対象
1. `zodスキーマ` の `.message()` / `.min()` / `.max()` 等
2. フォームの `errors` オブジェクト
3. `toast.error()` / `alert()` の引数
4. バリデーション関数の戻り値

### 記載例
| バリデーション | エラーメッセージ |
|----------------|-----------------|
| 必須入力 | `{項目名}を入力してください` |
| メール形式 | `有効なメールアドレスを入力してください` |
| 最大文字数超過 | `{N}文字以内で入力してください` |
| 数値範囲外 | `{N}〜{M}の範囲で入力してください` |
| 未選択 | `{項目名}を選択してください` |
| 複数エラーなし | `--` |

## 権限の判定ルール

`lib/utils/permissions.ts` の `PERMISSION_MATRIX` から判定する。

### 記載フォーマット
```
{対象ロール説明}（{権限レベル}）
```

### 権限レベル
| レベル | 意味 |
|--------|------|
| F | フルアクセス（全操作可能） |
| W | 閲覧 + 編集 |
| R | 閲覧のみ |
| C | 作成 + 自分の申請の閲覧 |
| X | アクセス不可 |

### 記載例
| 権限パターン | 記載 |
|-------------|------|
| 全ロール表示 | `全ロール` |
| 全ロール閲覧可（一部編集可） | `全ロール閲覧可 / SHIP系+病院管理者は編集可` |
| SHIP系のみ | `system_admin, org_default_1〜4のみ` |
| 病院系のみ | `hospital_sys_admin, hospital_office等` |
| 特定ロールのみ | `system_admin, hospital_sys_adminのみ` |
| 画面全体の制御 | `画面アクセス: {featureId}参照` |

### 要素レベルの権限
画面全体のアクセス権限と、要素個別の権限を区別する:
- **画面レベル**: 画面自体にアクセスできるかどうか → 備考欄に `画面アクセス権限: {featureId}` と記載
- **要素レベル**: 画面内の特定要素の表示/非表示、編集可否 → 権限欄に記載

## セレクタヒントの記載ルール

Playwright で要素位置を自動取得するためのCSSセレクタを備考欄に付与する。

### フォーマット
```
selector: {CSSセレクタ}
```

### 例
| 要素 | セレクタヒント |
|------|---------------|
| ログインボタン | `selector: button[type="submit"]` |
| メールアドレス入力 | `selector: input[type="email"]` |
| 施設選択プルダウン | `selector: [data-testid="facility-select"]` |
| 一覧テーブル | `selector: table` |
| タブ | `selector: [role="tab"]:nth-child(1)` |

### セレクタ優先順位
1. `data-testid` 属性
2. `role` 属性 + テキスト内容
3. `type` 属性（input系）
4. CSSクラス + テキスト（`button:has-text('...')`）
5. 構造的セレクタ（`.class > .child:nth-child(N)`）
